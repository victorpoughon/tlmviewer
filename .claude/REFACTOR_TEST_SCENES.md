# Plan: Self-contained test scenes co-located with elements

## Problem

All test scenes in `public/tests/` are JSON files generated by an external project (torchlensmaker). This means:
- Library development requires the external project to regenerate or update test data
- Adding a test for a new element type requires going outside the library
- The `surface-lathe` scenes contain hundreds of pre-computed `samples` coordinates that are opaque and hard to reason about

## Proposal

Reorganize `src/elements/` so each element type lives in its own subdirectory, alongside any element-specific utilities and a `scenes.ts` file that defines built-in test scenes for that element. The test harness imports from these scene files directly.

## Target directory layout

```
src/elements/
    AbstractSceneElement.ts       ← stays at top level (shared base)
    SurfaceBaseElement.ts         ← stays at top level (shared base)
    arrows/
        ArrowsElement.ts
        scenes.ts
    bcyl/
        BcylElement.ts
        scenes.ts
    box3D/
        Box3DElement.ts
        scenes.ts
    points/
        PointsElement.ts
        scenes.ts
    rays/
        RaysElement.ts
        scenes.ts
    surfaceLathe/
        SurfaceLatheElement.ts
        scenes.ts
    surfacePlane/
        SurfacePlaneElement.ts
        scenes.ts
    surfaceSphereR/
        SurfaceSphereRElement.ts
        scenes.ts
    surfaceSag/
        SurfaceSagElement.ts
        scenes.ts
```

Each `scenes.ts` exports an array of `{ name, data }` scene objects. A top-level `src/elements/scenes.ts` re-exports them all. The test harness imports this and renders the scenes alongside the existing JSON files from `public/tests/`.

### Step 2 — Add `scenes.ts` to each element directory

Create a `scenes.ts` in each element subdirectory. Each file exports a named array of scene objects:

```typescript
// src/elements/surfaceSag/scenes.ts
export const sagScenes = [
    { name: "surfaceSag/sag2D", data: { ... } },
    { name: "surfaceSag/sag3D", data: { ... } },
];
```

For `surface-lathe`, compute samples inline using a small math helper in the same file rather than storing pre-computed coordinate dumps:

```typescript
// src/elements/surfaceLathe/scenes.ts
function sphericalSamples(R: number, diameter: number, N: number) {
    const yMax = diameter / 2;
    return Array.from({ length: N }, (_, i) => {
        const y = -yMax + (i * 2 * yMax) / (N - 1);
        const x = R - Math.sign(R) * Math.sqrt(R * R - y * y);
        return [x, y];
    });
}

export const latheSccenes = [
    {
        name: "surfaceLathe/spherical2D",
        data: {
            mode: "2D", camera: "XY",
            data: [{
                type: "surface-lathe",
                samples: sphericalSamples(20, 10, 51),
                matrix: [[1, 0, 0], [0, 1, 0], [0, 0, 1]],
            }],
        },
    },
];
```

### Step 3 — Create the top-level scenes index

Create `src/elements/scenes.ts` that aggregates all element scenes into one export:

```typescript
// src/elements/scenes.ts
import { sagScenes } from "./surfaceSag/scenes.ts";
import { latheScenes } from "./surfaceLathe/scenes.ts";
// ... etc

export const builtinScenes = [
    ...sagScenes,
    ...latheScenes,
    // ...
];
```

### Step 4 — Wire into the test harness

Modify `test/test.ts` to import built-in scenes and add them to the test list alongside the JSON files from `public/tests/`. Built-in scenes are prefixed with `[builtin]` to distinguish them:

```typescript
import { builtinScenes } from "../src/elements/scenes.ts";

// After building the JSON file list:
for (const { name, data } of builtinScenes) {
    const li = document.createElement("li");
    const a = document.createElement("a");
    a.href = "#";
    a.textContent = `[builtin] ${name}`;
    a.addEventListener("click", (e) => {
        e.preventDefault();
        tlmviewer.embed(viewerElement, JSON.stringify(data));
    });
    li.appendChild(a);
    ul.appendChild(li);
}
```

## Notes

- `AbstractSceneElement.ts` and `SurfaceBaseElement.ts` stay at `src/elements/` root — they are shared base classes, not element types
- Element files that import from `SurfaceBaseElement` will need their relative import path adjusted (e.g. `../SurfaceBaseElement.ts` instead of `./SurfaceBaseElement.ts`)
- The `scenes.ts` files are only used in the test harness — they are not included in the library bundle
